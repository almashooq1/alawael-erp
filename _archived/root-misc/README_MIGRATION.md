3. عدّل `NEW_DB_CONFIG` بـ بيانات قاعدة البيانات الجديدة

## 🆘 حل المشاكل السريعة

### المشكلة: "Connection refused"
```
الحل:
1. تحقق من عنوان IP
2. تحقق من رقم المنفذ (3306 عادة)
3. تحقق من اسم المستخدم وكلمة المرور
4. تأكد أن قاعدة البيانات موجودة
```

### المشكلة: "Unknown table"
```
الحل:
1. فتش قاعدة البيانات القديمة
2. احصل على أسماء الجداول الفعلية
3. عدّل أسماء الجداول في الـ scripts
4. اختبر الاتصال مرة أخرى
```

### المشكلة: "Too many connections"
```
الحل:
1. انتظر 1-2 دقيقة
2. أعد محاولة الاتصال
3. أو قلل من عدد الاتصالات المتزامنة
```

### المشكلة: "Row count mismatch"
```
الحل:
1. شاهد عدد الصفوف في قاعدة البيانات القديمة
2. قارن مع عدد الصفوف في الجديدة
3. إذا كانت مختلفة:
   - تحقق من الأخطاء في السجل
   - استخدم rollback
   - اختبر مرة أخرى
```
