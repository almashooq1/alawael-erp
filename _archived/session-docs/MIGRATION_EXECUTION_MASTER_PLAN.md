```bash
# نقطة الحفظ الأخيرة قبل البدء
MIGRATION_TIME=$(date +%Y%m%d-%H%M%S)

mongodump \
  --uri="mongodb://old-server/alawael" \
  --out=/backups/pre-migration/$MIGRATION_TIME \
  --oplog

# تحقق من اكتمال النسخة
du -sh /backups/pre-migration/$MIGRATION_TIME
```

```bash
# Attendance: 987654
```

## 4️⃣ قائمة الفحص الشاملة

### قبل الهجرة (Pre-Migration)

```
التحضيرات:
  □ جميع الفرق مدربة على الهجرة والتراجع
  □ نصوص الهجرة والتراجع تم اختبارها
  □ خطوط الاتصال الطارئة مفعلة
  □ نقطة دعم 24/7 في الخدمة

البيانات:
  □ نسخة احتياطية كاملة موجودة
  □ تم التحقق من سلامة النسخة
  □ جميع البيانات معدودة وموثقة
  □ حسابات التحقق (Checksums) محفوظة

البيئة:
  □ قاعدة البيانات الجديدة فارغة
  □ الفهارس تم إنشاؤها
  □ Redis في حالة جيدة
  □ الذاكرة الحرة كافية (> 4GB)
  □ مساحة التخزين متوفرة (> 500GB)

الاختبارات:
  □ اختبار الاتصال نجح
  □ اختبار الهجرة الجاف نجح
  □ اختبار التراجع نجح
  □ اختبارات الأداء نجحت
```

### أثناء الهجرة (During Migration)

```
المراقبة:
  □ سجلات الهجرة يتم حفظها في الوقت الفعلي
  □ موارد النظام طبيعية
  □ لا توجد أخطاء غير متوقعة
  □ السرعة تقترب من التوقع

الفريق:
  □ جميع الفريق موجود وجاهز
  □ SSO قناة واحدة للاتصال
  □ تقارير الحالة كل 30 دقيقة
  □ فريق التراجع على استعداد
```

### بعد الهجرة (Post-Migration)

```
التحقق:
  □ جميع البيانات موجودة (100%)
  □ لا توجد بيانات مكررة
  □ المراجع صحيحة
  □ الأداء طبيعي

الاختبارات:
  □ اختبارات التطبيق تمر
  □ تسجيل الدخول يعمل
  □ التقارير تعمل
  □ المرفقات يمكن الوصول إليها

التوثيق:
  □ تم حفظ جميع السجلات
  □ تم توثيق الوقت الفعلي
  □ تم توثيق المشاكل والحلول
  □ تم تحديث الوثائق النهائية
```

## 6️⃣ سيناريوهات الطوارئ والتراجع

### السيناريو 1: البيانات الناقصة 🚨

```
الكشف:
  • عدد السجلات أقل من 95% من المتوقع
  • تنبيه فوري من نظام المراقبة

القرار (يجب أن يكون خلال دقيقتين):
  • تراجع فوري - تاريخ: _______ توقيع: _______

التنفيذ:
  bash execute-rollback.sh full "Missing data" "Manager Name"

التحقق:
  • البيانات استعادة بالكامل
  • النظام يعمل بشكل طبيعي
```

### السيناريو 2: الأداء البطيء ⚠️

```
الكشف:
  • وقت الاستجابة > 500ms
  • وسطيات التنبيهات تتجاوز الحد

التحليل (5 دقائق):
  • هل المشكلة في الفهارس؟
  • هل المشكلة في الموارد؟
  • هل يمكن تحسين الأداء؟

القرار:
  [ ] محاولة التحسين (30 دقيقة محدودة)
  [ ] تراجع فوري

إذا تراجع:
  bash execute-rollback.sh full "Performance issue" "Manager"
```

### السيناريو 3: المراجع المكسورة ❌

```
الكشف:
  • عدد السجلات اليتيمة > 100
  • أخطاء في سلامة البيانات

القرار الفوري:
  • تراجع فوري - تاريخ: _______ توقيع: _______

التنفيذ:
  bash execute-rollback.sh full "Referential integrity failure" "Manager"

التحليل اللاحق:
  • لماذا حدثت المشكلة؟
  • كيف نمنع تكرارها؟
  • هل نحتاج نسخة محسنة من الهجرة؟
```
