## 🚨 حالة الطوارئ - أول 5 دقائق

### إذا حدثت مصيبة:

#### ✋ توقف فوري:
```
1. أوقف جميع العمليات الجديدة
2. لا تشغل أي نسخ احتياطية أخرى
3. لا تحاول "إصلاح" أي شيء
```

#### 📢 أرسل تنبيهات فوري:
```
SLACK:   #database-alert (اذكر المشكلة بوضوح)
WHATSAPP: مجموعة الطوارئ الفنية
SMS:     رقم التنبيهات السريعة
```

#### 👥 اجمع الفريق:
```
من تحتاج:
  1. مدير المشروع → القرار النهائي
  2. DBA → تنفيذ التراجع
  3. DevOps → مراقبة البنية التحتية
  4. QA Lead → التحقق النهائي
```

## 🎬 التنفيذ الفعلي

### الخطوة 1: تشخيص سريع (30 ثانية)

```bash
# افتح Terminal واحد
cd /app/backend

# شغل فاحص النظام
node scripts/system-readiness.js

# شغل فحص الصحة السريع
curl http://localhost:3000/health

# اعرض السجلات الأخيرة
tail -f /var/log/alawael/migration.log
```

### الخطوة 2: القرار (2 دقيقة)

**ملء نموذج القرار:**
```
المشكلة المكتشفة:
  ________________________________

شدة المشكلة:
  [ ] بسيطة   [ ] متوسطة   [ ] حرجة

نوع التراجع:
  [ ] جزئي (مجموعة واحدة)
  [ ] كامل (كل شيء)

الموافقة:
  المديــر: ___________________
  الوقــت: ___________________
  التوقيــع: _________________
```

### الخطوة 3: التنفيذ (3 دقائق)

**Copy-Paste في Terminal:**

```bash
# للتراجع الجزئي:
cd /app/backend/scripts
bash execute-rollback.sh partial "data loss detected" "Ahmed Manager"

# أو للتراجع الكامل:
cd /app/backend/scripts
bash execute-rollback.sh full "database corruption" "Ahmed Manager"
```

### الخطوة 4: المراقبة (5 دقائق)

```bash
# افتح Terminal جديد وراقب:
tail -f /var/log/alawael/rollback-*.log

# يجب أن ترى:
[2026-02-25 15:00:00] Phase 1/5: Stopping application...
[2026-02-25 15:00:05] Phase 2/5: Dropping new database...
[2026-02-25 15:00:20] Phase 3/5: Restoring backup...
[2026-02-25 15:08:30] Phase 4/5: Rebuilding indexes...
[2026-02-25 15:10:00] Phase 5/5: Verifying and starting...

✅ ROLLBACK COMPLETED SUCCESSFULLY
```

### الخطوة 5: التحقق النهائي (1 دقيقة)

```bash
# اختبر الاتصال
curl http://localhost:3000/health
# يجب أن ترى: {"status":"ok"}

# اختبر تسجيل الدخول (إن وجد تطبيق اختبار)
npm test -- --testNamePattern="login"

# تحقق من عدد المستندات
node scripts/system-readiness.js
```

## 🎯 نقاط تذكر مهمة

### ✅ افعل:
- ✅ تواصل مباشر مع الفريق
- ✅ اتبع الخطوات بالترتيب
- ✅ وثق كل خطوة في السجل
- ✅ انتظر التأكيد قبل كل مرحلة
- ✅ احفظ السجلات الكاملة

### ❌ لا تفعل:
- ❌ لا تعدل قاعدة البيانات يدويا
- ❌ لا تشغل عمليات أخرى أثناء التراجع
- ❌ لا تأخذ قرارات بمفردك
- ❌ لا تغلق السجلات قبل الانتهاء
- ❌ لا تحاول "الإصلاح السريع"

## 🎓 الأدوار والمسؤوليات

### مدير المشروع
- اتخاذ قرار التراجع النهائي
- توافق من الإدارة العليا
- إشعار المستخدمين

### DBA
- فحص البيانات
- تنفيذ نص التراجع
- التحقق من النسخة الاحتياطية

### DevOps Engineer
- مراقبة البنية التحتية
- التحقق من الموارد
- إعادة تشغيل البيانات

### QA Lead
- اختبار التطبيق
- التحقق من الوظائف الرئيسية
- الموافقة على إعادة التشغيل
