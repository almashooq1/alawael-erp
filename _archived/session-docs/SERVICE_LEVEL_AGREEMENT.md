# 📋 اتفاقية مستوى الخدمة (SLA)
## Alawael Platform v1.0 - Service Level Agreement

**الإصدار**: 1.0  
**التاريخ**: 26 فبراير 2026  
**السريان**: من تاريخ الإطلاق حتى إشعار آخر

## 1. نظرة عامة

تحدد هذه الاتفاقية مستوى الخدمة المتوقع من منصة الأوائل (Alawael Platform) والتزامات فريق التشغيل تجاه المستخدمين.

---

## 2. التزامات التوفر (Availability Commitments)

### 2.1 الهدف الأساسي
```
┌─────────────────────────────────────────────┐
│          UPTIME TARGET                      │
│                                              │
│  المستهدف: 99.9% شهرياً                    │
│  يعني: أقل من 43 دقيقة توقف/شهر            │
│                                              │
└─────────────────────────────────────────────┘
```

### 2.2 مستويات التوفر

| المستوى | النسبة | التوقف المسموح/شهر | الحالة |
|---------|--------|---------------------|--------|
| 🟢 ممتاز | ≥ 99.9% | < 43 دقيقة | خدمة كاملة |
| 🟡 جيد | 99.5% - 99.9% | 22 - 43 دقيقة | مقبول |
| 🟠 متوسط | 99% - 99.5% | 43 - 7 ساعات | يحتاج تحسين |
| 🔁 ضعيف | < 99% | > 7 ساعات | غير مقبول |

### 2.3 استثناءات الحساب
```
لا تحتسب ضمن فترات التوقف:
├─ الصيانة المجدولة (يتم الإعلان مسبقاً)
├─ مشاكل مقدمي الخدمة الخارجيين (AWS, Azure)
├─ هجمات DDoS كبرى
├─ كوارث طبيعية (Force Majeure)
└─ مشاكل شبكة المستخدم
```

## 3. معايير الأداء (Performance Standards)

### 3.1 زمن الاستجابة (Response Time)

| نوع الطلب | الهدف p50 | الهدف p95 | الهدف p99 |
|-----------|-----------|-----------|-----------|
| صفحات ثابتة | < 50ms | < 100ms | < 200ms |
| API GET | < 100ms | < 200ms | < 500ms |
| API POST | < 150ms | < 300ms | < 600ms |
| تقارير | < 500ms | < 1s | < 2s |
| عمليات معقدة | < 1s | < 3s | < 5s |

### 3.2 معدل الأخطاء (Error Rate)
```
الهدف: < 0.1% (أقل من 1 خطأ لكل 1000 طلب)

حد التنبيه: > 0.5%
حد الطوارئ: > 1%
```

### 3.3 الإنتاجية (Throughput)
```
الحد الأدنى: 100 طلب/ثانية
الحد المتوقع: 500 طلب/ثانية
الحد الأقصى (Burst): 2000 طلب/ثانية
```

## 8. التعويضات (Service Credits)

### 8.1 سياسة التعويض
```
في حال عدم تحقيق SLA:

├─ Uptime 99.5% - 99.9%: لا تعويض
├─ Uptime 99% - 99.5%: تعويض 10% من الشهر
├─ Uptime 95% - 99%: تعويض 25% من الشهر
└─ Uptime < 95%: تعويض 50% من الشهر
```

### 8.2 شروط التعويض
```
├─ يجب تقديم الطلب خلال 30 يوم من الحادثة
├─ التعويض يُحسب من القيمة المدفوعة
├─ الحد الأقصى للتعويض: 50% من الرسوم الشهرية
└─ التعويض يُطبق كخصم على الفاتورة التالية
```

## 9. المراقبة والتقارير (Monitoring & Reporting)

### 9.1 المقاييس المراقبة
```
المقاييس التقنية:
├─ Uptime (توفر النظام)
├─ Response Time (زمن الاستجابة)
├─ Error Rate (معدل الأخطاء)
├─ Throughput (الإنتاجية)
├─ CPU Usage (استخدام المعالج)
├─ Memory Usage (استخدام الذاكرة)
├─ Disk I/O (قراءة/كتابة القرص)
└─ Network Latency (زمن استجابة الشبكة)
```

### 9.2 التقارير الدورية
```
├─ يومي: ملخص سريع للأداء
├─ أسبوعي: تقرير مفصل + مقارنة
├─ شهري: تقرير شامل + SLA compliance
└─ ربع سنوي: مراجعة وتحليل اتجاهات
```

## 10. الأمان والحماية (Security)

### 10.1 التزامات الأمان
```
├─ تشفير البيانات (at rest & in transit)
├─ مصادقة متعددة العوامل (MFA)
├─ فحص أمني دوري (كل 3 أشهر)
├─ تحديثات أمنية خلال 48 ساعة
├─ مراقبة التسلل 24/7
└─ تدريب الأمان للفريق
```

### 10.2 استجابة الحوادث الأمنية
```
├─ اكتشاف: < 1 ساعة
├─ احتواء: < 2 ساعة
├─ إعلام العملاء: < 24 ساعة (للحوادث الكبرى)
└─ تحليل ما بعد الحادثة: خلال 72 ساعة
```
