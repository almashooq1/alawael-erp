1. **~~بوّابة الاختبارات جزئية، والتغطية منخفضة.~~ ✅ عولج جزئياً (2026-06-15).**

   - **الدليل التجريبي:** شُغّلت كل الاختبارات اليدوية غير المُبوّبة (558 ملفاً) فنجح **9931/9932 اختباراً**. الفشل الوحيد كان قنبلة زمنية في `inspection-submission.test.js` (يثبّت `capturedAt` ويعتمد نافذة 30 يوماً افتراضية فيسقط خارجها مع تقدّم الساعة) — أُصلح بحقن `now`.

   - **إصلاح مُشغّل البوّابة (انحدار اكتُشف أثناء التحقق):** توسعة القائمة إلى 1599 ملفاً كسرت `run-sprint.js` على Windows بـ `spawn ENAMETOOLONG` (تعليق المُشغّل ادّعى خطأً أنه محدود فقط بـ ARG_MAX بالميغابايت؛ لكن `CreateProcess` على Windows يحدّ كامل سطر الأوامر بـ32767 حرفاً). أُعيد كتابة المُشغّل ليُمرّر قائمة الملفات إلى واجهة jest البرمجية داخل العملية (`@jest/core` runCLI) عند تجاوز السقف — فلا تلمس القائمة سطر أوامر النظام إطلاقاً؛ مع إبقاء مسار `spawn` المُثبت للقوائم القصيرة ودعم `--shard` كما هو. **تحقّق نهائي:** `npm run test:sprint -- --shard=1/8` نفّذ 200 مجموعة / 4497 اختباراً، كلها خضراء (exit 0) في ~158ث عبر المسار المتوازي الحقيقي.
