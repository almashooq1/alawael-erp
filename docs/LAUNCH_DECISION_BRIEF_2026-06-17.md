## ٤. مسار ١٠ دقائق لأول استخدام حقيقي

> طوبولوجيا الإنتاج للخطوات الإدارية: `ssh root@72.60.84.56` · `/home/alawael/app/backend` · pm2 `alawael-api` (مستخدم `alawael`) · بعد أي تغيير env: `pm2 restart alawael-api --update-env`.

1. **دخول** بمستخدم مدير حقيقي → تأكّد أن الدخول يعمل (لا وضع تجريبي).
2. **أنشئ فرعاً + موظفين حقيقيين** عبر الواجهة (PII حقيقي — لا يُزرع). الأدوار في `docs/architecture/PRODUCTION_CUTOVER_W356_W370.md`.
3. **سجّل مستفيداً حقيقياً** عبر النموذج العربي → يجب أن يحفظ بلا 500 (جسر W926 يقبل التسميات العربية).
4. **سجّل جلسة** له مع تقدّم هدف → تظهر في مركز الجلسات والتحليلات (إسقاط W1240).
5. **افتح `/launch-readiness`** في web-admin → راقب الحُكم يبقى GO ببيانات حقيقية.

تحقّق فوري في أي لحظة (قراءة فقط / آمن): `npm run launch:readiness`. تحقّق فعلي عميق: `npm run smoke:launch-spine` و`npm run smoke:clinical-spine` (يُنشئان ويحذفان بياناتهما).

## ٥. لماذا التوقّف عن بناء المزيد الآن (توصية صريحة)

واقع الإنتاج (لقطة W1235): ~190 سطحاً مبنياً، معظمها فارغ؛ 18 مستفيداً / 13 مستخدماً، أغلبها عرض. **العائق ليس الميزات ولا الأدوات — بل التبنّي.** بناء مزيد من الكود قبل أول استخدام حقيقي = قيمة متناقصة (نحرس فراغاً). أعلى رافعة: **اطلق بمركز واحد حقيقي**، ودع الاستخدام يكشف الفجوات الفعلية — عندها يصبح للإصلاح أثرٌ حقيقي لا تخميني.

## ٦. ماذا أفعل أنا بعد الإطلاق

اللحظة التي يدخل فيها مستخدم حقيقي، يصبح عملي عالي الرافعة فوراً:

- أراقب أخطاء Winston الحيّة وأُصلح ما يصطدم به مستخدم فعلي (لا ما أتخيّله).

- جاهز أيضاً الآن لأي من: أداة وسم العرض (القرار أ) · تدقيق أمني قبل-PII · حاجة مسمّاة تحددها.
