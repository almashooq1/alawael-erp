# 38a — بطاقة جلسة AAC (A4 — اطبعها)

> اطبع هذه الصفحة. امسكها في يدك اليسرى أثناء الجلسة. اكتب الملاحظات على دفتر ورقي منفصل في يدك اليمنى. **لا laptop**. القاعدة الذهبية: **اصمت**.
>
> الجمهور المفضّل: **معالج نطق (SLP)**. البديل: OT يعمل مع مستفيد غير ناطق، أو والد PECS Phase 1-3.

---

## ١. السيناريو الأولي — اقرأ هذا، ثم اصمت

> "عندك مستفيد جديد، 5 سنوات، توحد، غير ناطق، يدخل المركز اليوم لأول مرة. الأم تَطلب منك أن تشرح لها 'كيف سنُدخله نظام PECS'. افتح النظام كأنك ستُدخل ملفه الآن."

اكتب في دفترك ٣ أعمدة:

| ما يفعله (الإجراء) | ما يقوله (حرفياً) | ما يلاحظ (تردد، تحديق، نقرة خاطئة) |
| ------------------ | ----------------- | ---------------------------------- |
|                    |                   |                                    |

دَع له **٢٠ دقيقة** بلا توجيه. راقب خاصة: هل يَبحث في navigation؟ يَكتب beneficiaryId يدوياً؟ يَفتح ملف المستفيد أولاً ويَبحث عن "AAC tab"؟

---

## ٢. الخمس أسئلة المغلقة — للدقائق العشر الأخيرة

اطرحها بالترتيب. لا تُعِد الصياغة. اقبل أي إجابة.

| #   | السؤال                                                                      | الإجابة التي تَكشف فجوة                               |
| --- | --------------------------------------------------------------------------- | ----------------------------------------------------- |
| ١   | "المستوى الاستقبالي vs التعبيري — الفرق واضح من العناوين، أم احتجت تتذكّر؟" | "كلاهما نفس الشيء" → التصنيف ضائع                     |
| ٢   | "الـ9 استراتيجيات شركاء — ضغطت بعضها بدون تأكيد، أم تركتها؟"                | "ضغطت كلها" أو "تركتها كلها" → الـUI لا تُمكّن قرار   |
| ٣   | "PECS Phase 1→2: كم محاولة ناجحة تحتاج لتسجيل الانتقال؟ وأين تكتبها؟"       | إجابة فضفاضة → criteriaMet نَص حر غير كافٍ            |
| ٤   | "الـskip-gate يَرفض القفز 1→3. صادفت حالة كان فيها هذا خطأ من النظام؟"      | "نعم، طفل تعلّم phase 2 خارج المركز" → نحتاج override |
| ٥   | "لو فقدنا الـsymbol library تماماً، ماذا ستفقد؟"                            | "لا شيء، أستخدم BoardMaker" → احذف المكتبة            |

## ٣. العلامات الحمراء — اكتب نجمة ⭐ بجانبها فوراً

- ⭐ يَفتح `/aac` ثم يُغلق ويَبحث عن AAC في صفحة beneficiary → **IA خطأ**
- ⭐ يَتوقف عند الـ5 modality options ⇐ التصنيف يحتاج شرح
- ⭐ يَكتب beneficiaryId يدوياً → نَنقص beneficiary picker
- ⭐ "هل سيُطبع هذا للأهل؟" → use case ورقي لم نَبنه
- ⭐ يَستخدم Excel/Word لتتبع PECS بشكل موازٍ → **خسرنا**

---

## ٤. ما لا تَفعله أبداً

- لا تَفتح `/aac` نيابة عنه. لو لم يَصل لها = signal.
- لا تَشرح PECS له. لو لم يَعرفها = هو ليس المستخدم.
- لا تَعِد بحذف الـsymbol library في الجلسة.
- لا تَدافع عن Rowland terminology لو وَجدها غامضة.

---

## ٥. بعد الجلسة — خلال ١٥ دقيقة

١. **اكتب فوراً** قبل أن تَنسى الـnonverbal cues
٢. حدّد **٣ ملاحظات سلوكية** (فعَلها أو لم يَفعلها)، ليس آراء
٣. ابعث **رسالة شكر واحدة** — بلا أسئلة إضافية
٤. **انتظر ٤٨ ساعة** قبل أي تعديل

---

## ٦. القرار بعد ٤٨ ساعة — اختر **واحداً**

| المسار                    | الإجراء                                              |
| ------------------------- | ---------------------------------------------------- |
| **A — AAC يبقى ويُمدَّد** | ابنِ AacBoard builder + symbol seeder + image upload |
| **B — AAC يبقى، يتبسّط**  | احذف symbol library، اختصر partner strategies إلى ٤  |
| **C — AAC يُؤرشَف**       | الـUI تَبقى لكن لا تُروَّج. المعالج لا يَستخدمها     |

> الأرشفة قرار محترم، ليس فشلاً.
