# 40a — بطاقة جلسة BIP Tracking (A4 — اطبعها)

> اطبع هذه الصفحة. امسكها في يدك اليسرى. اكتب على دفتر منفصل في يدك اليمنى. **لا laptop**. القاعدة الذهبية: **اصمت**.
>
> جلستان: **مشرف فرع** (المستخدم الفعلي) أولاً، ثم **BCBA** (المُصدِّق المهني).
>
> السؤال المحوري: **"هل سيَفتح المشرف `/bip-tracking` كل خميس صباحاً؟"** بدون discipline أسبوعية، الـworkflow ميت بَغض النظر عن جمال الـUI.

---

## ١. السيناريو الأولي — المشرف

> "خميس صباحاً. مستفيد على BIP نشطة منذ ٦ أسابيع. مديرك السريري سيَسألك الإثنين: 'هل الفريق طَبَّق الخطة؟' افتح النظام كأنك ستُجيب."

اكتب:

| ما يفعله | ما يقوله | ما يلاحظ |
| -------- | -------- | -------- |
|          |          |          |

دَع له **١٥ دقيقة**. الـmoments الحرجة:

1. هل وَجد `/bip-tracking/[fbaId]` من الـnavigation أم احتاج رابطاً؟
2. كم معياراً كَتب في نموذج الـfidelity؟ ٣ (الافتراضي) أم زاد؟
3. هل أَدخل قيمة لكل معيار، أم تَجاهل بعضها؟
4. هل ضَغط حفظ بدون barrier حتى لو لاحَظ مشكلة؟

## ٢. الـ٥ أسئلة — المشرف

| #   | السؤال                                                                                                 | الإجابة التي تَكشف فجوة                                    |
| --- | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------- |
| ١   | "الـ٣ معايير كانت فارغة. ماذا كَتبت؟ ولماذا هذه؟"                                                      | معايير عامة جداً ("الفريق يَتبع") → نَحتاج template seed   |
| ٢   | "NA checkbox — استخدمته؟ متى يَكون منطقياً؟"                                                           | "إن لم أَستطع المراقبة كَتبت 0" → الـformula تَنحاز للأسفل |

| ٤   | "barrier=staff_turnover يَرفع supervisor review تلقائياً. الـstaff_undertrained أيضاً — كثير أم قليل؟" | "كثير noise" أو "قليل، يَفلت حالات"                        |
| ٥   | "diagnosis 'hypothesis_likely_wrong' عَرَضت. فَتحت FBA لتُراجع أم تَجاهلت؟"                            | **أهم سؤال**. "تَجاهلت" → diagnosis = ornament لا workflow |

## ٣. السيناريو الأولي — BCBA (جلسة منفصلة)

> "حالة BIP نشط ٨ أسابيع. fidelity الأسبوع الماضي 75%، فعالية مستقرة. افتح الـdiagnosis."

### الـ٥ أسئلة — BCBA

| #   | السؤال                                                                                          | الفجوة                                              |
| --- | ----------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| ١   | "الـ4-bucket: التَصنيف يَتفق مع تَقديرك المهني؟"                                                | "لا" → اطلب السبب المحدد                            |
| ٢   | "fidelity 75% = concerning. النظام يَدفعك للتَصرف، أم مَجرد يُخبرك؟"                            | "يُخبر فقط" → نَحتاج recommended action لكل status  |
| ٣   | "الـsnapshot يَجمد baseline من FBA. قَيّمت قيمته؟ تَفهم أن تَعديل FBA لاحقاً لا يُغيّر الماضي؟" | "لم أَلاحظ" → audit-trail UI غير ظاهر               |
| ٤   | "تَراجع التَكرار = تَحسُّن النتيجة. كان واضحاً، أم احتجت أن تُفكّر؟"                            | "احتجت أن أُفكّر" → tooltip "للأهداف التَخفيضية..." |
| ٥   | "replacement: frequency + successRate كافٍ، أم تَحتاج independence level؟"                      | "أحتاج independence" → ابنِ الـselect (data exists) |

## ٤. العلامات الحمراء ⭐

- ⭐ المشرف لا يَفتح `/bip-tracking` من نفسه — يَطلب رابطاً → **IA broken**
- ⭐ يَفتح، يَرى diagnosis card، يُغلق دون تَسجيل قراءة → **dashboard ≠ workflow**
- ⭐ "أحبّ أن أَكتب ملاحظاتي في الجلسة وتُسجَّل تلقائياً" → **خسرنا** (يَكتب في Word)
- ⭐ يَملأ كل المعايير بـ100% بسرعة → **social desirability bias**
- ⭐ يَطلب طباعة fidelity check لتَوقيع المعالج → نَحتاج PDF + signature
- ⭐ BCBA: "لا أَستخدم 4-bucket، أَستخدم خبرتي" → النص ornament

---

## ٥. ما لا تَفعله

- لا تُعطِ مشرفاً لائحة معايير جاهزة لتَملأ بسرعة. أنت تَختبر هل **هو** يَعرف ماذا يَكتب.
- لا تَدافع عن 4-bucket لو BCBA قَلَّل منها. اشكر واكتب.
- **لا تَسأل** "هل ستَستخدمها كل أسبوع؟" — إجابة مهذبة عديمة القيمة.
- اسأل بدلاً: **"الأسبوع الماضي، كم مرة فَتحت `/bip-tracking`؟"** الإجابة الصادقة على الفعل الماضي تَكشف المستقبل.

## ٧. القرار بعد ٤٨ ساعة

| المسار                         | الإجراء                                                       |
| ------------------------------ | ------------------------------------------------------------- |
| **A — auto-trigger**           | scheduled reminder كل ٧ أيام لكل FBA نشط (notification/email) |
| **B — قَيِّد للحالات الحرجة**  | UI تَظهر فقط لـFBA priority=high لتَخفيف overhead             |
| **C — يُؤرشَف، أَبقِ FBA فقط** | weekly fidelity يَحتاج BCBA-led setting، نُعيد بعد ٦ أشهر     |

> الجواب الصادق غالباً: **B**. القياس الأسبوعي يَنجح فقط في BCBA-led settings.
