### 2. mongoose 9 hook compatibility shim

mongoose 9 stopped passing `next` to document `pre('save')` / `post`
hooks. 92+ models in the codebase still use the legacy
`function (next) { ...; next(); }` shape, which now throws
`TypeError: next is not a function` at save-time. Surfaced as a 500
on `POST /api/admin/invoices` — the only path the test suite actually
exercises through `Invoice.create`.

Two-pronged fix:

- **`Invoice.pre('save')`** rewritten to the modern no-arg shape
  (commit `2a90bea3`).
- **Single-file shim in `backend/config/mongoose.plugins.js`**
  (commit `c165f25a`) — patches `Schema.prototype.pre/post` once at
  module-load. Any hook function that declares a `next` parameter is
  wrapped in an adapter that re-injects a real callback. Modern
  hooks (no parameter) and parallel hooks (next, done) bypass
  untouched. Idempotent via `__legacyShimmed` flag. Verified with a
  4-case probe (sync legacy / async legacy / next(err) / modern) —
  all four pass.

The shim protects every legacy model without rewriting 92 files.
Commit message records the diagnostic + alternatives considered.

### 7. Broken-requires sweep — 503 → 6 false positives + drift guard

A static-scan probe (strip comments, walk the tree, resolve every
`require('../...')` against the filesystem) found 503 broken require
statements across 18 files at the start of the session. Most were
wrapped in try/catch so production ran with silently-disabled
features.

Eleven cleanup commits brought the count to 6 — every remaining
match is a documented false positive:

| File                                       | Why it's not a real broken require                                                                                                             |
| ------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| `scripts/migrations/secure-routes.js` (×5) | The `require(...)` strings are _templates_ the script inserts into target route files at runtime; correct from the route's POV (one level up). |
| `tests/unit/check_app.root.test.js` (×1)   | Legacy test runner outside the canonical `__tests__/` directory, never run by `npm test`.                                                      |

Biggest single win: archiving `scripts/verify-ddd-platform.js`
(7006 LOC, 476 broken requires, fully retired DDD verifier) to
`_archived/scripts/` — commit `9caacd75`. Removed `ddd:verify` from
package.json + the corresponding row in `scripts/README.md`.

The 11 cleanup commits are documented row-by-row in the memory
entry `project_broken_requires_followup_2026-04-28.md` (ledger from
oldest to newest, with the per-site verdict for each file touched).

**Drift guard (commit `85b27f60`):**
`backend/__tests__/no-broken-requires.test.js` walks every backend
`.js` file, strips comments, and resolves every literal
`require('./...')` / `require('../...')` against the filesystem.
Wired into `test:sprint` + `test:ops-subsystems` + `test:drift`. A
future PR that introduces a typo'd or stale relative import gets
caught at CI time instead of accumulating into another cleanup
sprint. Allowlist for two documented false positives (the migration
script's string templates + a legacy stub outside `__tests__/`).
