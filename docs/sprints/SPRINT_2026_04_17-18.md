## Silent bugs fixed

The new API smoke tests caught **2 production-critical silent bugs** in
the existing route registry:

1. `_registry.js:86` — destructured `{ router }` from `safeRequire`'s
   empty-Router fallback → `undefined` handler → `app.use` threw and
   aborted `mountAllRoutes` → **~15 new admin routes silently never
   mounted** in any environment missing `routes/purchasing.routes.unified`.

2. `_registry.js:111` — same pattern on `inventory.routes.unified`.

Both fixed with defensive destructure:

```js
const x = (safeRequire(...) || {}).router || express.Router();
```

Without the test suite, these would have shipped to staging and
mysteriously 404'd in production until someone stumbled on them.

## Architecture highlights

### Uniform adapter interface

All 10 government adapters expose the same surface:

```ts
interface GovAdapter {
  MODE: 'mock' | 'live';
  verify(params): Promise<{ status, mode, message?, ... }>;
  testConnection(): Promise<{ ok, mode, latencyMs?, error? }>;
  getConfig(): { provider, mode, configured, missing?, timeoutMs };
}
```

Adding a new gov integration (e.g. Tameeni for insurance) is a single
file in `backend/services/` + one line in `gov-integrations.routes.js`.

### Production-grade GOSI

The first adapter was upgraded beyond the mock pattern and serves as
the template for hardening others:

- Token caching with 10s expiry buffer
- Circuit breaker: 5 failures/60s → open for 120s cooldown
- Per-request `AbortController` timeout
- Auto-retry on network error + 401 (forced token refresh)
- Structured logging of circuit state

### ZATCA Phase-2 pipeline

```
Invoice (DB)
  → buildEnvelope() — UUID + ICV + PIH + TLV-QR
  → buildUblInvoice() — UBL 2.1 XML
  → canonicalize() — strip prolog/comments/whitespace
  → digest() — SHA-256 base64
  → SignedInfo → RSA-SHA256 signature via production key
  → injectSignature() — wrap with UBLExtensions + ds:Signature
  → base64 XML → Fatoora API → reporting | clearance
  → response.status → invoice.zatca.zatcaStatus
```

The in-test signature verification (using `crypto.generateKeyPairSync`

- `crypto.createVerify`) **cryptographically proves** the pipeline
  is correct without needing real ZATCA credentials.

## What's NOT done (deliberately)

- **Real vendor onboarding** with GOSI/SCFHS/Fatoora/etc. — business
  step requiring commercial registration numbers, signed agreements,
  and CSR/CSID certificates.
- **App Store / Play Store deployment** — requires Apple + Google
  developer accounts and app bundles not yet signed.
- **Load testing at production scale** — all modules have been unit +
  integration tested; stress testing happens in staging.

Everything marked "live" in this sprint is code-complete; only
environment variables need to change to flip from mock to real.

## Post-sprint follow-ups (4.0.1 → 4.0.4)

After the 2-day core sprint shipped, a follow-on batch added the
production-readiness layer that the initial scope deliberately
deferred. Tracked as semver patch releases; full details in
[`CHANGELOG.md`](../../CHANGELOG.md).

### 4.0.1 — Cost-protection rate limiter

Token-bucket per provider + per-actor sub-cap. Paid adapters
(Absher/NPHIES/Fatoora) get tighter defaults; env overrides via
`{PROVIDER}_RL_CAPACITY` etc. 429 responses carry `retryAfterMs` +
`scope` so clients can back off intelligently. Mobile client
auto-retries once using the server-suggested delay.

### 4.0.2 — Ops dashboards (admin UI)

Three specialist pages — rate limits · PDPL audit · unified matrix —
plus an embeddable `IntegrationsHealthBadge` for the main admin
landing page. All polling-based (10–60s depending on signal volatility).

### 4.0.3 — Reliability + observability

Shared `adapterCircuitBreaker` factory + named registry. Wired into
GOSI (refactored to shared code, byte-identical) and the 3 other paid
adapters. Prometheus text-format metrics endpoint at
`/api/health/metrics/integrations` — 9 metric families × 10 providers
including counters (`gov_adapter_calls_total`) and a latency
histogram. CSV export for the PDPL audit trail (UTF-8 BOM for
Excel/Arabic).

### 4.0.4 — Docs, runbooks, CI widening

3 Alertmanager rule groups (7 rules) + 3 on-call runbooks cover every
alert severity. Grafana dashboard with 12 panels (4 stat tiles +
timeseries + state-timeline + SLI panels). Go-live doc gains an
Observability section. Sprint CI gate widened from 122 tests across
4 jobs to 191 tests across 5 jobs — `ops-artifacts` suite
structurally validates the Grafana JSON + Alertmanager YAML so
dashboard drift is caught at PR time instead of reload time. CLI
ops-status script for SSH / cron consumption.

### 4.0.5 — Correlation IDs + deploy gate + DX polish

The final 4.0.x patch closes the last PDPL gap (cascaded-call
correlation) and blocks the one class of prod incident that kept
showing up in runbooks — someone flipping `*_MODE=live` without
the secrets.

- `AdapterAudit.correlationId` indexed field wired through audit
  logger from `req.id` (existing X-Request-Id middleware). New
  `GET /admin/adapter-audit/by-correlation/:id` + UI cascade dialog
  in `AdminAdapterAudit` turn DPO DSAR into a 2-click flow.
- `backend/scripts/preflight.js` — deploy gate that exits non-zero
  if any live adapter is missing env vars. Three modes (TTY / JSON /
  CI-stderr). 7 exit-code contract tests spawned as child processes.
- Operator ergonomics: `gov:status` CLI (exit 0/1/2 for cron),
  `OPERATIONS.md` one-page front door, root `Makefile` wrapping
  14 targets with auto-generated `make help` menu.
