  ```bash
  git worktree add /tmp/fix <child-branch>

  git push --force-with-lease ; gh pr edit <n> --base master
  ```

كل شيء **يعمل بأمان دون أي علم جديد** عدا التالي. فعّل بالترتيب (الأقل خطرًا أولًا):

| العلم                                       | الموجة  | الافتراضي | الأثر عند التفعيل                                                         |
| ------------------------------------------- | ------- | --------- | ------------------------------------------------------------------------- |
| (لا شيء)                                    | 1–5,7,8 | —         | السياق + الملاحظات + التحويل + المواعيد + KPIs + الإعدادات تعمل فور الدمج |
| `ENABLE_WHATSAPP_CAMPAIGN_SCHEDULER=true`   | 6       | **OFF**   | إطلاق آلي ساعتيّ للحملات المجدوَلة المستحقّة                              |
| `ENABLE_WHATSAPP_POST_SESSION_SUMMARY=true` | 10 · M5 | **OFF**   | **رسالة آلية للولي عند اكتمال كل جلسة.** ⚠️ راجع التواتر للحضور اليومي    |
| `ENABLE_WHATSAPP_COMPLAINT_RESOLVED=true`   | 11 · M6 | **OFF**   | رسالة آلية للولي عند حلّ شكوى مرتبطة بمستفيد                              |
| `ENABLE_WHATSAPP_EVENT_BINDINGS=true`       | 12 · M5 | **OFF**   | **موزّع عام** للروابط التي يُنشئها المسؤول من واجهة «أتمتة الرسائل» (حدث→رسالة بلا كود) |
| `ENABLE_WHATSAPP_APPOINTMENT_REMINDERS=true` | 13 (W1525) | **OFF** | **كنّاسة كل 15 دقيقة تُسلّم تذكيرات المواعيد المستحقّة** للولي (تقليل الغياب). idempotent + بفلتر موافقة |
| `ENABLE_WHATSAPP_REMINDER_AUTO_ENQUEUE=true` | 13 (W1527/W1534) | **OFF** | يُسجّل التذكيرات آليًا عند **حجز** الموعد ويُلغيها عند **إلغائه** (بلا لمس كود الحجز). مستقل عن علم التسليم |

**اعتماديات Meta المسبقة (موجودة من قبل — ليست من هذه الموجات):** `WHATSAPP_API_TOKEN` + `WHATSAPP_PHONE_ID` + `WHATSAPP_WEBHOOK_SECRET` + `WHATSAPP_VERIFY_TOKEN` (راجع `SETUP_AND_OTP.md`). بدونها تبقى عمليات الإرسال في وضع stub (آمن للاختبار).

> **سلامة المُجدوِل:** `runCampaign` يضبط `status='running'` ← التشغيل المزدوج آمن (لا إرسال مكرّر)؛ كل مستلم يمرّ بفلتر الموافقة + تحديد المعدّل. مُعطَّل افتراضيًا = أثر صفري.

**⚠️ سلامة الأتمتة (الأعلام 10/11/12 — رسائل آلية للأسر):** كل الأحداث مُعطّلة افتراضيًا وخاملة تمامًا حتى التفعيل المتعمَّد. عند التفعيل: (1) كل رسالة تمرّ ببوابة موافقة `WhatsAppConsent.canMessage`؛ (2) تتحوّل stub تلقائيًا إن لم تُفعَّل اعتمادات Meta؛ (3) **اختبر أولًا برقم اختبار** قبل التفعيل في الإنتاج؛ (4) للحضور اليومي، راجع تواتر «ملخّص ما بعد الجلسة» (رسالة لكل جلسة قد تكون كثيفة) — فعّله انتقائيًا أو أضف تصفية قبل التوسّع. حلّ الولي عبر `whatsappGuardianResolver` المشترك (ولي شرعي → راعٍ رئيسي → **والد/ولي/جدّ** → أول برقم؛ تفضيل الولي المسؤول أُضيف في W1519).

**⚠️ إرسال مزدوج (العلم 12 مقابل 10/11):** العلمان الثابتان (10/11) والموزّع القابل للتهيئة (12) يغطّيان نفس الأحداث (`session.completed` / `complaint.resolved`). استخدم **إمّا علمًا ثابتًا وإمّا رابطًا** للحدث الواحد — لا كليهما (وإلّا رسالتان). كلاهما OFF افتراضيًا، فلا تداخل ما لم تُفعّلهما معًا عمدًا. الروابط تُدار من `/admin` → اتصالات → واتساب → **أتمتة الرسائل** (أو `POST /api/v1/whatsapp-automation/bindings`).

**📅 تذكيرات المواعيد (العلم 13 — تقليل الغياب، W1525/W1527/W1534):** نظام من مرحلتين مستقلّتَين بعلمين منفصلين:

1. `ENABLE_WHATSAPP_REMINDER_AUTO_ENQUEUE=true` — **المُنتِج**: عند حجز موعد (حدث `appointment.booked`) يُسجَّل تذكيرَا 24س + ساعتين تلقائيًا؛ وعند إلغاء الموعد (`appointment.cancelled`) تُلغى تذكيراته المعلّقة (فلا تُرسَل لموعد مُلغى). idempotent — لا يُكرّر الصفّ.
2. `ENABLE_WHATSAPP_APPOINTMENT_REMINDERS=true` — **المُسلِّم**: كنّاسة كل 15 دقيقة تُرسل التذكيرات المستحقّة (حجز ذرّي بزيادة المحاولات → لا إرسال مزدوج)؛ كل تذكير بفلتر موافقة + حدّ محاولات (3). تُحلّ بيانات الولي عبر `whatsappGuardianResolver` المشترك.

> **خطوات التفعيل:** فعّل `…AUTO_ENQUEUE` أولًا (يبدأ التسجيل عند الحجوزات الجديدة)، اختبر برقم اختبار، ثم فعّل `…APPOINTMENT_REMINDERS` (يبدأ التسليم). راقب `GET /api/v1/whatsapp-reminders/stats` (مُرسَل/معلّق/فشل/مُلغى) من لوحة المؤشرات. **التحكم اليدوي:** `POST /whatsapp-reminders/enqueue` لموعد + `POST /whatsapp-reminders/dispatch` للتشغيل الفوري.

**📊 مؤشرات وأدوات الواجهة (بلا أعلام — تعمل فور دمج رِبو):** صحّة التفاعل (الأسر الصامتة → إعادة تواصل، W1526) + لوحة تحكم واتساب (مركز قيادة، W1530) + مُختبِر المساعد الذكي (W1531) + رابط/QR المحادثة (W1532) + فاحص حدّ الإرسال (W1533) — كلها للقراءة (عدا تشغيل التذكيرات اليدوي) ومعزولة بالفرع. الوصول من `/admin` → اتصالات → واتساب → **لوحة التحكم**.

## 4) قائمة تحقّق الإطلاق (Cutover Checklist)

1. **ادمج الخلفية** (#633 ثم #635) في `main` → ينشر تلقائيًا (راجع runbook النشر).
2. **ادمج سلسلة الواجهة** بالترتيب #154→#158 (Merge commit، لا squash).
3. **تحقّق من القراءة-فقط فورًا** (لا أعلام لازمة):
   - افتح محادثة في الـInbox → يظهر **شريط السياق** (مستفيد/خطة/أهداف/جلسات/فواتير).
   - أضف **ملاحظة داخلية** → تظهر بالكاتب والوقت، وليست في خيط واتساب.
   - على جلسة قادمة: زرّا **«تغيّب»/«إلغاء»** → تتحدّث حالة الجلسة (حدث `session.no_show`/`session.cancelled` → الخط الزمني).
   - افتح **«مؤشرات الأثر»** → اعتماد + غياب (مع/بدون واتساب) + أهداف + NPS.
4. **أنشئ حملة تجريبية** (مسودة) → شغّلها يدويًا (`/run`) على مجموعة صغيرة → راجع المقاييس (مُرسَل/مؤهَّل/محجوب).
5. **فعّل المُجدوِل** أخيرًا: `ENABLE_WHATSAPP_CAMPAIGN_SCHEDULER=true` + أعد تشغيل pm2 (إعادة قراءة `.env`) → أنشئ حملة **مجدوَلة** بوقت قريب → تأكّد إطلاقها آليًا خلال الساعة.

## 5) التحقّق السريع عبر API (curl)

```bash
# سياق المستفيد لمحادثة
GET  /api/v1/whatsapp/conversations/:id/context
# ملاحظة داخلية
POST /api/v1/whatsapp/conversations/:id/notes        { "text": "…" }
# تحويل مُدقَّق + قائمة الموظفين القابلين (للمُنتقي)
POST /api/v1/whatsapp/conversations/:id/transfer     { "staffId": "…", "reason": "…" }
GET  /api/v1/whatsapp/assignable-staff               # موظفو الفرع النشطون
# مزامنة موعد
POST /api/v1/whatsapp/conversations/:id/appointment-action { "sessionId":"…", "action":"no_show" }
# حملات
GET/POST /api/v1/whatsapp/campaigns ; POST /api/v1/whatsapp/campaigns/:id/run
POST /api/v1/whatsapp/campaigns/run-due              # إطلاق المستحقّة (يدوي/خارجي)
# مؤشرات الأثر + الموافقات (الإعدادات)
GET  /api/v1/whatsapp/analytics/rehab-outcomes?windowDays=90
GET  /api/v1/whatsapp/consent/:phone ; POST .../consent/:phone/opt-in|/opt-out
```

كل النقاط معزولة بالفرع؛ معرّف من فرع آخر ← 404 نظيف.

## 7) ملاحظات الموثوقية والامتثال

- **العزل:** كل قراءة/إجراء عبر `effectiveBranchScope`/`branchFilter` (W1407/W1412)؛ لا `req.branchId` (W269h).
- **عدم التكرار:** الموجات تعيد استخدام قناة واتساب القائمة (whatsappService، بوت FSM، 7 نماذج) ونماذج النواة بـ `ref` فقط — لا نموذج محادثة/رسالة جديد.
- **مؤشرات M8 ارتباطية لا سببية** — مذكور صراحةً في الواجهة والكود.
- **التفويض النظيف (M3):** مزامنة المواعيد تمرّ عبر facade نطاق الجلسات (`cancelSession`/`markNoShow`) فتُصدر الأحداث القانونية ولا تتجاوز منطق النطاق.
- **الحوكمة:** كل موجة شُحنت بحارس انحراف (static + pure) + بوابات pre-push السبع خضراء + worktree معزول.
