الخطوة 3️⃣: نشر الخدمات
──────────────────────────────────

**Option A: باستخدام Docker Compose (الأسهل)**

```bash
docker-compose -f docker-compose.production.yml up -d
```

**Option B: باستخدام Kubernetes (الأفضل)**

```bash
kubectl apply -f k8s/production/
```

**Option C: باستخدام الخدمات السحابية**

- **AWS**: استخدم Elastic Container Service (ECS)
- **Azure**: استخدم Azure Container Instances (ACI)
- **GCP**: استخدم Cloud Run

الوقت المتوقع: 5-10 دقائق

🆘 **خطة الطوارئ (في حالة المشاكل):**

═══════════════════════════════════════════════════════════════════

المشكلة: النظام لا يستجيب

الحل السريع:
1. أعد تشغيل الخدمات: docker restart alawael-backend
2. تحقق من الأخطاء: docker logs alawael-backend
3. أعد البيانات: mongorestore --archive=backup.archive
4. العودة إلى النسخة السابقة إذا لزم الأمر

───────────────────────────────────────────────────

المشكلة: الأداء بطيء جداً

الحل السريع:
1. تحقق من استخدام CPU/Memory
2. أعد توازن الحمل: docker-compose scale backend=3
3. امسح الرسائل القديمة من قاعدة البيانات
4. فعّل الذاكرة المؤقتة (Redis)

───────────────────────────────────────────────────

المشكلة: أخطاء في الاتصال

الحل السريع:
1. تحقق من اتصال MongoDB
2. تحقق من متغيرات البيئة
3. تحقق من قوائم حريق الجدار
4. أعد تشغيل خدمة الشبكة
