# PHASE 5 OPS PLAYBOOKS

## 1. Playbook: ارتفاع زمن الاستجابة

**المؤشرات | Triggers:** P95 > 140ms لمدة 15 دقيقة

**الإجراءات | Actions:**

1. التحقق من لوحات المراقبة (CPU/Memory/DB)
2. مراجعة أعلى الاستعلامات بطئاً
3. زيادة الكاش أو ضبط TTL
4. توسيع الخدمة المتأثرة
5. التحقق من انخفاض P95 بعد 30 دقيقة

---

## 2. Playbook: ارتفاع معدل الخطأ

**المؤشرات | Triggers:** Error Rate > 0.20% لمدة 10 دقائق

**الإجراءات | Actions:**

1. مراجعة سجلات الأخطاء
2. تحديد الخدمة المتأثرة
3. تفعيل rollback أو feature flag
4. تنفيذ Hotfix إذا لزم
5. تحديث RCA بعد الاستقرار

---

## 3. Playbook: ضغط دعم العملاء

**المؤشرات | Triggers:** تذاكر الدعم > 300/أسبوع

**الإجراءات | Actions:**

1. زيادة تغطية فريق الدعم
2. تحديث قاعدة المعرفة
3. إصدار تحديثات توعوية للمستخدمين
4. تحليل الأسباب المتكررة وإصلاحها

---

## 4. Playbook: مشكلات التوسع

**المؤشرات | Triggers:** CPU > 75% لمدة 20 دقيقة

**الإجراءات | Actions:**

1. تفعيل Auto-Scaling
2. التحقق من سياسات التوسع
3. إضافة موارد مؤقتة
4. تحديث خطة السعة

---

## 5. Playbook: حادث أمني

**المؤشرات | Triggers:** Security Alert High/Critical

**الإجراءات | Actions:**

1. تصعيد فوري إلى Security Lead
2. عزل الخدمة المتأثرة إذا لزم
3. تفعيل خطة الطوارئ الأمنية
4. إصدار Patch أمني
5. توثيق RCA خلال 24 ساعة
