## 2. إجراءات المراقبة اليومية | Daily Monitoring Procedures

**جدول المراقبة | Monitoring Schedule:**

- 08:00: مراجعة لوحة المؤشرات
- 12:00: مراجعة الأداء والإنذارات
- 16:00: مراجعة السعة والتوسع
- 20:00: مراجعة الحوادث اليومية

**قائمة تحقق سريعة | Quick Checks:**

- [ ] Response Time P95 < 120ms
- [ ] Error Rate < 0.12%
- [ ] Uptime >= 99.99%
- [ ] CPU < 70% و Memory < 80%
- [ ] 0 تنبيهات حرجة غير محلولة

---

## 3. إجراءات الحوادث | Incident Procedures

### 3.1 تصنيف الحوادث | Severity Classification

| الشدة | Severity | الوصف              | Description | زمن الاستجابة | SLA |
| ----- | -------- | ------------------ | ----------- | ------------- | --- |
| Sev-1 | Critical | انقطاع كامل        | 15 min      |
| Sev-2 | High     | خدمة رئيسية متأثرة | 30 min      |
| Sev-3 | Medium   | تدهور جزئي         | 2 hours     |
| Sev-4 | Low      | تأثير محدود        | 24 hours    |

### 3.2 خطوات الاستجابة | Response Steps

1. تحديد الحادثة وتأكيد التأثير
2. إشعار المسؤول حسب مستوى الشدة
3. تطبيق إجراءات التخفيف الفورية
4. توثيق التحديثات كل 30 دقيقة (Sev-1/2)
5. إنهاء الحادثة وبدء تحليل السبب الجذري

---

## 4. إجراءات الأداء | Performance Optimization

- مراجعة أعلى 5 استعلامات بطيئة أسبوعياً
- تحسين إعدادات الكاش حسب الحمل
- ضبط سياسات Auto-Scaling عند تجاوز 70% CPU
- متابعة مؤشرات P95/P99 بشكل يومي

---

## 5. إجراءات التوسع | Scaling Procedures

| الشرط                      | Trigger             | الإجراء           | Action |
| -------------------------- | ------------------- | ----------------- | ------ |
| CPU > 70% لمدة 15 دقيقة    | Scale Out           | إضافة مثيل تلقائي |
| Memory > 80% لمدة 15 دقيقة | Scale Out           | إضافة ذاكرة/مثيل  |
| Error Rate > 0.2%          | Scale & Investigate | توسيع + تحليل     |
| P95 > 140ms                | Performance Review  | إعادة ضبط الكاش   |
